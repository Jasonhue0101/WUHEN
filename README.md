## ⚙️ 隐私保护配置

为了保障隐私安全，我已启用以下 GitHub 安全特性：

### 1. 邮件地址保护
- [x] **Keep my email addresses private**: 提交代码时使用 GitHub 提供的匿名 ID 邮件。
- [x] **Block command line pushes that expose my personal email**: 阻止任何包含私密邮箱的推送请求。

### 2. 交互限制
* **不接收**来自非关注者的私密报告（如适用）。
* **自动屏蔽** 包含敏感信息的 Issue 或 Comment（通过 GitHub Actions 过滤）。

---

## 🛠️ 如何管理您的可见性？

如果您是本项目的贡献者或访问者，建议检查您的 [GitHub Settings](https://github.com)：

1. **Profile 页面**：设置是否向所有人显示您的组织成员身份。
2. **Emails 页面**：开启 `Keep my email addresses private`。
3. **Security 页面**：启用 **2FA (双重身份验证)** 保护您的资料不被非法篡改。

## 💡 最佳实践建议

> 💡 **核心原则：最小必要原则**

*   **求职者**：建议将“当前职级”和“核心技能”设为公开，将“目前薪资”和“联系电话”设为私密。
*   **企业内网用户**：建议将“办公座机”和“所属部门”设为组织内公开，方便跨部门协作。
*   **社交达人**：建议开启“水印保护”，防止他人下载并盗用您的公开照片。

## ⚠️ 安全提示

1.  **定期自检**：建议每季度检查一次隐私设置，确保权限符合当前需求。
2.  **第三方授权**：当您使用社交账号登录外部应用时，请注意该应用是否具有“读取完整个人资料”的权限。
3.  **敏感数据**：平台绝不会要求您将支付密码、银行卡号等信息设为“公开”。
